## Typography

Font choices signal professionalism. Wrong fonts undermine credibility before anyone reads a word.

### Font Categories for Documents

| Category | Use | Examples |
|----------|-----|----------|
| **Serif** | Body text in printed documents | Times New Roman, Georgia, Garamond, Cambria |
| **Sans-serif** | Headings, on-screen reading | Arial, Calibri, Helvetica, Inter |
| **Monospace** | Code, financial tables | Consolas, Courier New |

### Font Size Guidelines

| Element | Size Range | Notes |
|---------|------------|-------|
| **Body text** | 11-12pt | 12pt for formal/legal; 11pt for dense content |
| **Headings (Level 1)** | 14-16pt | Bold, clear differentiation from body |
| **Headings (Level 2)** | 12-14pt | Bold or semi-bold |
| **Headings (Level 3)** | 11-12pt | Bold or italic |
| **Footnotes/Sources** | 9-10pt | Clearly subordinate |
| **Captions** | 10-11pt | Below figures/tables |

### Professional Font Pairings

| Context | Headings | Body |
|---------|----------|------|
| **Traditional/Legal** | Times New Roman Bold | Times New Roman |
| **Modern Corporate** | Calibri Bold | Calibri |
| **Consulting** | Arial Bold | Times New Roman or Georgia |
| **Financial** | Georgia Bold | Georgia |
| **Academic** | Garamond Bold | Garamond |

### Typography Rules

1. **Two fonts maximum** - One for headings, one for body (or single font family)
2. **Consistent sizes** - Same hierarchy throughout document
3. **Line spacing** - 1.15 to 1.5 for body text (1.5 for draft review)
4. **Line length** - 60-80 characters optimal (avoid full 8.5" lines)
5. **Adequate contrast** - Black text on white; avoid light grays
6. **No decorative fonts** - Comic Sans, Papyrus, etc. destroy credibility

### Common Mistakes

| Mistake | Fix |
|---------|-----|
| ALL CAPS for emphasis | Use bold instead |
| Underlining (web style) | Use bold or italic |
| Justified text creating "rivers" | Use left-aligned (ragged right) |
| Tiny font to fit more content | Edit content instead |
| Multiple font sizes for emphasis | Pick one emphasis style |
| Colored text for headings | Keep black; use size/weight for hierarchy |

## Page Layout

Professional documents follow predictable structural patterns.

### Standard Margins

| Document Type | Margins | Notes |
|---------------|---------|-------|
| **Legal/Court filings** | 1" all sides | Often mandated |
| **Business documents** | 1" to 1.25" | Standard Word default |
| **Consulting deliverables** | 1" to 1.5" | More white space = more premium |
| **Academic** | 1" all sides | Double-spaced body |

### Page Numbering

| Style | When to Use |
|-------|-------------|
| **Bottom center** | Short documents, memos |
| **Bottom right** | Reports, multi-section documents |
| **Page X of Y** | Legal, formal submissions |
| **Section-page (2-5)** | Long reports with multiple sections |

### Headers and Footers

**Header content options:**
- Document title (left or center)
- Section name (left)
- Company name/logo (right)
- Date (right)
- Classification/Confidentiality (left)

**Footer content options:**
- Page numbers (center or right)
- Document version (left)
- Date/Author (left)
- Confidentiality notice (center)

### Common Layouts

**Single-column (Standard):**
```
┌────────────────────────────────────────────────────────────┐
│                                                            │
│  [Header: Document Title]                                  │
│                                                            │
│  ────────────────────────────────────────────────────────  │
│                                                            │
│  Body text flows full width. Standard for memos,           │
│  letters, contracts, and most business documents.          │
│                                                            │
│  Paragraph text continues here...                          │
│                                                            │
│  ────────────────────────────────────────────────────────  │
│                                                            │
│                        [Page 1]                            │
└────────────────────────────────────────────────────────────┘
```

**Two-column (Reports):**
```
┌────────────────────────────────────────────────────────────┐
│                                                            │
│              Executive Summary Title                       │
│                                                            │
│  ────────────────────────────────────────────────────────  │
│                                                            │
│  Column 1 text        │      Column 2 text                 │
│  continues here       │      continues here                │
│  with more content    │      with more content             │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

## Visual Hierarchy

Every page needs clear visual organization. Guide the reader's eye.

### The Hierarchy Tools

| Tool | Effect |
|------|--------|
| **Size** | Larger = more important |
| **Weight** | Bold stands out |
| **Position** | Top reads first |
| **Spacing** | Isolation creates importance |
| **Numbering** | Signals structure and sequence |

### F-Pattern Reading

Like slides, readers scan documents in an F-pattern:

```
┌─────────────────────────────────────────────────────────────┐
│ ████████████████████████████████████████████  ← Title scan  │
│ ██████████████████████████████████████                      │
│ ████████████████████████████  ← Section head scan           │
│ ████████████████████                                        │
│ ████████████  ← Quick vertical scan                         │
│ ████████                                                    │
│ ██████                                                      │
│ ████████████████████████████  ← Another section             │
└─────────────────────────────────────────────────────────────┘
```

**Implications:**
- Put key conclusions at the start of documents and sections
- Use headings to create scanning points
- Don't bury important information in long paragraphs
- First sentences of paragraphs carry disproportionate weight

### Creating Hierarchy in Documents

**Bad (No hierarchy):**
```
The project analysis shows that we should proceed with Option B.
Option B offers 15% cost savings compared to Option A while
maintaining quality standards. The implementation timeline is 6
months with a team of 4 FTEs. Risk factors include vendor
dependency and integration complexity. We recommend Board
approval at the next meeting.
```

**Good (Clear hierarchy):**
```
RECOMMENDATION: Proceed with Option B

Key Benefits:
• 15% cost savings vs. Option A
• Quality standards maintained
• 6-month implementation timeline

Resource Requirements:
• Team: 4 FTEs
• Budget: $500K

Risk Factors:
• Vendor dependency (mitigated by backup vendor)
• Integration complexity (addressed in Phase 2 plan)

Next Step: Board approval at December meeting
```

## White Space

Empty space is a design element, not wasted space.

### Why White Space Matters

- **Readability** - Prevents wall-of-text overwhelm
- **Focus** - Isolates important content
- **Professionalism** - Signals confidence and quality
- **Scanability** - Creates entry points for readers

### Types of White Space

**Macro white space:** Margins, space between sections
- At least 1" margins on all sides
- Extra space above major headings (12-18pt)
- Clear separation between document sections

**Micro white space:** Line spacing, paragraph spacing
- 1.15-1.5 line spacing for body text
- 6-12pt space after paragraphs
- Don't use both first-line indent AND paragraph spacing

### Paragraph Spacing Options

| Style | First-line Indent | Space After |
|-------|-------------------|-------------|
| **Block style** | None | 6-12pt |
| **Traditional** | 0.5" | None |
| **Mixed** | None | 6pt, extra before headings |

**Block style** (no indent, space between paragraphs) is standard for business documents. Traditional style (indented first line, no space) is for books and academic papers.

### Common White Space Mistakes

| Mistake | Fix |
|---------|-----|
| Content edge-to-edge | Add generous margins |
| No space between sections | Add extra spacing before major headings |
| Cramming to fit on one page | Either edit content or use two pages |
| Multiple blank lines for spacing | Use paragraph spacing instead |

## Headings

Headings are the skeleton of your document. They enable scanning and signal structure.

### Heading Levels

A document typically uses 3-4 levels maximum:

```
1. FIRST-LEVEL HEADING (or "Heading 1")
   Body text under first level...

   1.1 Second-Level Heading
   Body text under second level...

       1.1.1 Third-Level Heading
       Body text under third level...
```

### Heading Formatting by Level

| Level | Typical Style |
|-------|---------------|
| **Level 1** | Bold, 14-16pt, extra space above |
| **Level 2** | Bold, 12-14pt, space above |
| **Level 3** | Bold or italic, 11-12pt |
| **Level 4** | Italic, same size as body |

### Numbering Systems

| System | Example | When to Use |
|--------|---------|-------------|
| **Decimal** | 1, 1.1, 1.1.1 | Technical docs, legal |
| **Legal** | I, A, 1, a | Contracts, formal documents |
| **None** | Just formatted text | Short memos, letters |

### Heading Rules

1. **Never orphan a heading** - At least 2-3 lines of text must follow before page break
2. **Parallel structure** - All headings at same level use same grammatical form
3. **Descriptive, not generic** - "Budget Implications" not "Section 3"
4. **Keep headings short** - Ideally under 8 words
5. **Use Word Styles** - Don't manually format; use built-in styles for consistency and navigation

### Parallel Structure Examples

**Bad (mixed forms):**
- Implementation Steps
- How We Will Measure Success
- Risks
- Budget

**Good (parallel nouns):**
- Implementation Steps
- Success Metrics
- Risk Factors
- Budget Requirements

**Good (parallel gerunds):**
- Implementing the Solution
- Measuring Success
- Managing Risks
- Allocating Budget

## Consistency

Consistency builds credibility. Inconsistency creates cognitive load and signals carelessness.

### What Must Be Consistent

| Element | Rule |
|---------|------|
| **Fonts** | Same typefaces throughout |
| **Sizes** | Same hierarchy everywhere |
| **Heading style** | Same format at each level |
| **Spacing** | Same paragraph spacing throughout |
| **Numbering** | Same system in all lists |
| **Date format** | Pick one: January 15, 2025 or 1/15/2025 |
| **Number format** | Pick one: 1,000 or 1000; $1M or $1 million |

### Use Word Styles

The most important consistency tool is Word's Styles feature:

1. **Define styles once** - Set font, size, spacing for each heading level
2. **Apply styles everywhere** - Never manually format headings
3. **Modify in one place** - Change a style, and all instances update
4. **Enable navigation** - Styles create automatic Table of Contents and Navigation Pane

### Consistency Checklist

- [ ] All body text same font and size
- [ ] All Level 1 headings identical format
- [ ] All Level 2 headings identical format
- [ ] All bulleted lists use same bullet style
- [ ] All numbered lists use same number format
- [ ] Page numbers consistent placement
- [ ] Headers/footers consistent throughout
- [ ] Margins identical all pages

### Hierarchy Checklist
- [ ] Key message at the start
- [ ] Headings break up long content
- [ ] First sentences of paragraphs carry main points
- [ ] Important content not buried

### Consistency Checklist
- [ ] Word Styles used throughout
- [ ] Formatting identical for same elements
- [ ] Date and number formats consistent
- [ ] Spacing uniform throughout

### The 10-Second Test

A reader glancing at your document for 10 seconds should be able to:
1. Understand the document's purpose from title/heading
2. Identify the main sections via headings
3. Find the key conclusion or recommendation
4. Know where to look for details

If they can't, your hierarchy needs work.
